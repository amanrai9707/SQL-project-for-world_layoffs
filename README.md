# 🌍 World Layoffs Data Cleaning & EDA using SQL


1. Project Overview

This project focuses on cleaning and analyzing a real-world layoffs dataset using SQL.

The project is divided into two parts:

- Data Cleaning – Removed duplicate records, handled missing values, standardized inconsistent data, and converted data types.
- Exploratory Data Analysis (EDA) – Analyzed layoff trends by company, country, industry, year, and funding to gain meaningful business insights.

---

2. Tools Used

- MySQL
- SQL

---

3. Dataset

The dataset contains information about company layoffs, including:

- Company
- Industry
- Country
- Date
- Total Employees Laid Off
- Percentage Laid Off
- Funding Raised

---

 4. Data Cleaning

The following cleaning steps were performed:

- Created a staging table
- Removed duplicate records
- Standardized inconsistent values
- Handled missing values
- Converted the date column to `DATE` format
- Removed unnecessary records

---

5. Exploratory Data Analysis

Performed SQL queries to answer questions such as:

- Which companies had the highest layoffs?
- Which industries were most affected?
- Which countries experienced the most layoffs?
- How did layoffs change over the years?
- Which companies had 100% layoffs?
- What is the monthly rolling total of layoffs?

7. Key Insights

- The technology industry experienced some of the largest layoffs.
- Several startups laid off their entire workforce.
- The United States recorded the highest number of layoffs.
- Layoffs peaked during specific years and varied across industries.
